제20강: 성능 최적화

Obsidian의 성능을 극대화하고 대규모 볼트를 효율적으로 관리하는 방법을 배웁니다

중급 60분 최적화

1. 성능 최적화 개요

왜 성능 최적화가 중요한가?

Obsidian은 로컬 파일 시스템을 기반으로 작동하므로, 볼트가 커질수록 성능 문제가 발생할 수 있습니다. 적절한 최적화를 통해 수천, 수만 개의 노트를 가진 볼트에서도 빠르고 안정적인 성능을 유지할 수 있습니다.

주요 성능 지표

< 0.5s
노트 열기 시간
< 2s
검색 응답 시간
< 1GB
메모리 사용량
> 60fps
UI 프레임레이트

성능에 영향을 미치는 요소

  • 볼트 크기: 노트 수, 첨부파일 크기
  • 플러그인: 활성화된 플러그인 수와 복잡도
  • 테마와 CSS: 커스텀 스타일의 복잡도
  • 백링크와 태그: 연결의 복잡도
  • 검색 인덱스: 인덱싱된 콘텐츠 양
  • 실시간 미리보기: 렌더링 부하

  • 정기적으로 아카이브 폴더로 이동
  • 첨부파일은 별도 폴더에 중앙화

대용량 파일 처리

  • 이미지 최적화: WebP 포맷 사용, 최대 너비 1920px
  • PDF 분할: 50MB 이상 PDF는 페이지별로 분할
  • 외부 저장소 활용: 대용량 파일은 클라우드 링크로 대체
  • 첨부파일 정리: 사용하지 않는 첨부파일 정기적 삭제

const lazyLoadImages = () => {
    const imageObserver = new IntersectionObserver((entries) => {
        entries.forEach(entry => {
            if (entry.isIntersecting) {
                const img = entry.target;
                img.src = img.dataset.src;
                imageObserver.unobserve(img);
            }
        });
    });
    
    document.querySelectorAll('img[data-src]')
        .forEach(img => imageObserver.observe(img));
};

자동 성능 모니터링

// 성능 모니터링 플러그인 스니펫
class PerformanceMonitor {
    constructor() {
        this.metrics = {
            fileOpenTimes: [],
            searchTimes: [],
            memoryUsage: []
        };
    }
    
    trackFileOpen(fileName, duration) {
        this.metrics.fileOpenTimes.push({
            file: fileName,
            duration: duration,
            timestamp: new Date()
        });
        
        // 평균 계산
        const avg = this.metrics.fileOpenTimes
            .reduce((sum, item) => sum + item.duration, 0) 
            / this.metrics.fileOpenTimes.length;
            
        if (avg > 500) {
            this.showWarning('파일 열기 속도가 느립니다');
        }
    }
    
    checkMemoryUsage() {
        if (performance.memory) {
            const usage = performance.memory.usedJSHeapSize / 1048576;
            this.metrics.memoryUsage.push(usage);
            
            if (usage > 1000) {
                this.showWarning('메모리 사용량이 높습니다');
            }
        }
    }
}

FAQ

Obsidian이 점점 느려지는 이유는 무엇인가요?
주요 원인으로는 볼트 크기 증가, 플러그인 누적, 캐시 파일 증가, 검색 인덱스 비대화 등이 있습니다. 정기적인 최적화 작업으로 해결할 수 있습니다.
어느 정도 크기의 볼트까지 효율적으로 관리할 수 있나요?
적절한 최적화를 통해 10,000개 이상의 노트와 수 GB의 첨부파일을 포함한 볼트도 원활히 관리할 수 있습니다. 핵심은 구조화와 정기적인 유지보수입니다.
플러그인을 많이 설치하면 얼마나 느려지나요?
플러그인마다 다르지만, 일반적으로 10-15개의 플러그인은 큰 영향이 없습니다. 중요한 것은 플러그인의 품질과 최적화 정도입니다. 필요한 플러그인만 활성화하고 정기적으로 업데이트하세요.
검색이 느려졌을 때 어떻게 해결하나요?
검색 인덱스를 재구축하고, 검색 제외 폴더를 설정하며, 검색 쿼리를 최적화하세요. 특히 아카이브나 첨부파일 폴더를 검색에서 제외하면 크게 개선됩니다.
모바일에서 성능을 개선하는 방법은?
모바일에서는 필수 플러그인만 활성화하고, 동기화할 폴더를 제한하며, 이미지 크기를 최적화하세요. 또한 오프라인 사용을 위해 필요한 파일만 다운로드하도록 설정하세요.
