🔍 Obsidian 쿼리의 힘

Dataview와 DataviewJS를 활용하면 노트를 데이터베이스처럼 활용할 수 있습니다. 복잡한 쿼리로 원하는 정보를 즉시 찾고, 동적으로 업데이트되는 뷰를 만들 수 있습니다.

실시간 쿼리 데모

Dataview 쿼리

```dataview TABLE WITHOUT ID file.link as "노트", author as "작성자", rating as "평점", dateformat(date, "yyyy-MM-dd") as "작성일" FROM "Books" WHERE rating >= 4 SORT rating DESC, date DESC LIMIT 5 ```

노트 작성자 평점 작성일

📚 딥 워크 칼 뉴포트 ⭐⭐⭐⭐⭐ 2024-01-15

🚀 고급 Dataview 쿼리 기법

📊 집계 함수

데이터를 그룹화하고 통계를 계산합니다.

```dataview TABLE WITHOUT ID category as "카테고리", length(rows) as "노트 수", sum(rows.words) as "총 단어 수", round(average(rows.rating), 2) as "평균 평점" FROM "Content" GROUP BY category SORT length(rows) DESC ```

🔗 관계 쿼리

노트 간의 연결 관계를 분석합니다.

```dataview TABLE WITHOUT ID file.link as "노트", length(file.inlinks) as "들어오는 링크", length(file.outlinks) as "나가는 링크", file.inlinks as "연결된 노트" FROM "" WHERE length(file.inlinks) > 3 SORT length(file.inlinks) DESC ```

📅 날짜 연산

날짜 기반 필터링과 계산을 수행합니다.

```dataview TABLE WITHOUT ID file.link as "작업", due as "마감일", (due - date(today)).days as "남은 일수" FROM "Tasks" WHERE due > date(today) AND due <= date(today) + dur(7 days) SORT due ASC ```

🎨 시각적 표현

데이터를 시각적으로 표현합니다.

```dataview TABLE WITHOUT ID file.link as "프로젝트", "🟩" * (progress/10) + "⬜" * (10-progress/10) as "진행률", progress + "%" as "%" FROM "Projects" WHERE progress SORT progress DESC ```

🔍 복합 조건

여러 조건을 조합하여 정확한 결과를 얻습니다.

```dataview LIST FROM "Daily" WHERE contains(tags, "important") AND date >= date(today) - dur(30 days) AND (mood >= 7 OR productivity >= 8) AND !completed SORT date DESC ```

📈 동적 차트

데이터를 차트로 시각화합니다.

```dataviewjs const pages = dv.pages('"Daily"') .where(p => p.date >= dv.date("today-30d")) .map(p => ({date: p.date, mood: p.mood})) .sort(p => p.date); dv.header(3, "30일 기분 추이"); dv.paragraph(pages.map(p => `${p.date.toFormat("MM-dd")}: ${"⭐".repeat(p.mood)}` ).join("\n")); ```

💻 DataviewJS로 무한한 가능성 열기

JavaScript의 강력함을 활용하여 복잡한 로직과 커스텀 뷰를 구현할 수 있습니다.

1. 커스텀 대시보드

```dataviewjs // 오늘의 대시보드 const today = dv.date("today"); const tasks = dv.pages('"Tasks"').where(p => p.due); dv.header(2, "📅 오늘의 할 일"); const todayTasks = tasks.where(p => p.due.equals(today)); if (todayTasks.length > 0) { dv.taskList(todayTasks.file.tasks); } else { dv.paragraph("✨ 오늘은 예정된 작업이 없습니다!"); } dv.header(2, "🔥 긴급 작업"); const urgentTasks = tasks .where(p => p.priority === "urgent" && !p.completed) .sort(p => p.due); dv.table(["작업", "마감일", "상태"], urgentTasks.map(p => [ p.file.link, p.due.toFormat("MM-dd"), p.status || "진행중" ]) ); ```

2. 습관 추적기

```dataviewjs // 최근 30일 습관 추적 const habits = ["운동", "독서", "명상", "글쓰기"]; const days = 30; const pages = dv.pages('"Daily"') .where(p => p.date >= dv.date(`today-${days}d`)) .sort(p => p.date, 'desc'); dv.header(2, "🎯 30일 습관 추적"); // 습관별 달성률 계산 habits.forEach(habit => { const completed = pages.where(p => p.habits && p.habits[habit]).length; const percentage = Math.round((completed / days) * 100); const bar = "█".repeat(Math.floor(percentage / 5)) + "░".repeat(20 - Math.floor(percentage / 5)); dv.paragraph(`**${habit}**: ${bar} ${percentage}% (${completed}/${days}일)`); }); ```

3. 네트워크 분석

```dataviewjs // 가장 연결이 많은 노트 찾기 const allPages = dv.pages('""'); const pageConnections = allPages .map(p => ({ page: p.file.link, connections: p.file.inlinks.length + p.file.outlinks.length, inlinks: p.file.inlinks.length, outlinks: p.file.outlinks.length })) .sort(p => p.connections, 'desc') .limit(10); dv.header(2, "🌐 네트워크 허브 노트"); dv.table( ["노트", "총 연결", "들어오는", "나가는"], pageConnections.map(p => [ p.page, p.connections, p.inlinks, p.outlinks ]) ); ```

4. 읽기 통계

```dataviewjs // 독서 통계 분석 const books = dv.pages('"Books"').where(p => p.status === "completed"); const thisYear = books.where(p => p.finished_date && p.finished_date.year === dv.date("today").year); dv.header(2, "📚 올해의 독서 통계"); // 월별 독서량 const monthlyReading = {}; thisYear.forEach(book => { const month = book.finished_date.monthLong; monthlyReading[month] = (monthlyReading[month] || 0) + 1; }); dv.paragraph(`**총 ${thisYear.length}권 완독**`); dv.list(Object.entries(monthlyReading) .map(([month, count]) => `${month}: ${count}권`)); // 장르별 분포 const genres = {}; thisYear.forEach(book => { if (book.genre) { genres[book.genre] = (genres[book.genre] || 0) + 1; } }); dv.header(3, "장르별 분포"); dv.list(Object.entries(genres) .sort((a, b) => b[1] - a[1]) .map(([genre, count]) => `${genre}: ${count}권`)); ```

✨ 쿼리와 자동화 모범 사례

1. 성능 최적화

대량의 노트를 다룰 때 쿼리 성능을 최적화하세요.

  • FROM 절에서 특정 폴더로 범위 제한
  • 복잡한 계산은 DataviewJS로 캐싱
  • 자주 사용하는 쿼리는 별도 노트에 저장
  • WHERE 조건을 구체적으로 작성

2. 에러 처리

자동화 스크립트에는 항상 에러 처리를 포함하세요.

try { const result = await someOperation(); if (!result) { new Notice("작업이 취소되었습니다."); return; } } catch (error) { console.error("오류 발생:", error); new Notice(`오류: ${error.message}`); }

3. 일관된 명명 규칙

메타데이터와 파일명에 일관된 규칙을 적용하세요.

  • 날짜: YYYY-MM-DD 형식 통일
  • 상태값: 영문 소문자 (todo, doing, done)
  • 폴더 구조: 계층적이고 직관적으로
  • 태그: 단수형, 소문자, 하이픈 연결

4. 점진적 구축

한 번에 모든 것을 자동화하려 하지 마세요.

  1. 수동으로 워크플로우 확립
  2. 반복적인 부분 식별
  3. 단순한 자동화부터 시작
  4. 피드백을 통한 개선
  5. 점진적으로 복잡도 증가

5. 문서화

자동화 시스템을 문서화하여 유지보수를 쉽게 하세요.

  • 각 쿼리의 목적과 사용법 설명
  • 템플릿 변수 목록 작성
  • 매크로 워크플로우 다이어그램
  • 트러블슈팅 가이드 작성

6. 백업과 버전 관리

자동화 스크립트와 템플릿을 안전하게 관리하세요.

  • Git으로 템플릿 버전 관리
  • 중요 스크립트는 별도 백업
  • 변경 전 테스트 환경에서 검증
  • 롤백 계획 수립

❓ 자주 묻는 질문

Dataview 쿼리가 느려요. 어떻게 최적화하나요?

다음 방법들로 성능을 개선할 수 있습니다:

  • FROM 절에서 특정 폴더나 태그로 범위 제한
  • 복잡한 계산은 DataviewJS에서 한 번만 수행하고 캐싱
  • LIMIT으로 결과 개수 제한
  • 인덱싱이 완료될 때까지 기다리기
  • 불필요한 파일 속성 접근 최소화
Templater 스크립트에서 에러가 발생해요

Templater 디버깅 방법:

  • 개발자 콘솔(Ctrl+Shift+I)에서 에러 메시지 확인
  • tp.system.prompt() 대신 기본값으로 테스트
  • 스크립트를 작은 단위로 나누어 테스트
  • Templater 설정에서 "Enable System Commands" 확인
  • 사용자 스크립트 폴더 경로가 올바른지 확인
QuickAdd와 Templater 중 어떤 것을 사용해야 하나요?

용도에 따라 선택하세요:

  • QuickAdd: 빠른 캡처, 간단한 매크로, UI 중심 워크플로우
  • Templater: 복잡한 템플릿, 프로그래밍 로직, 동적 콘텐츠 생성
  • 두 플러그인은 함께 사용할 수 있으며, QuickAdd에서 Templater 템플릿을 호출할 수 있습니다
버튼이 작동하지 않아요

버튼 플러그인 문제 해결:

  • 버튼 코드 블록이 정확히 ```button으로 시작하는지 확인
  • YAML 형식이 올바른지 확인 (들여쓰기, 콜론 뒤 공백)
  • 명령어 이름이 정확한지 확인 (대소문자 구분)
  • JavaScript 버튼의 경우 코드 문법 오류 확인
  • 읽기 모드에서만 버튼이 표시됨
자동화가 너무 복잡해졌어요. 어떻게 관리하나요?

복잡한 자동화 시스템 관리 방법:

  • 기능별로 모듈화하여 관리
  • 각 자동화의 목적과 사용법을 문서화
  • 정기적으로 사용하지 않는 자동화 정리
  • 복잡한 로직은 단계별로 분해
  • 테스트용 볼트에서 먼저 실험
  • 버전 관리로 변경 이력 추적
