제14강: 메타데이터와 속성 활용

노트에 구조화된 데이터를 추가하여 강력한 지식 관리 시스템을 구축하세요

📊 메타데이터란?

메타데이터는 '데이터에 대한 데이터'로, 노트의 속성 정보를 구조화하여 저장하는 방법입니다. Obsidian에서는 YAML Front Matter를 통해 메타데이터를 관리합니다.

YAML Front Matter 예시

---
title: "프로젝트 관리 시스템 구축"
tags: [프로젝트, 개발, 시스템]
status: 진행중
priority: 높음
created: 2024-01-15
due: 2024-02-28
assignee: 김개발
progress: 65
related:
- "[[기술 스택 선정]]"
- "[[아키텍처 설계]]"
---

🔤 메타데이터 속성 타입

📝 텍스트 (Text)

단순 문자열 값을 저장합니다. 제목, 설명, 작성자 등에 사용됩니다.

title: "나의 첫 번째 노트" author: "홍길동" description: "메타데이터 학습용 예제"

🔢 숫자 (Number)

정수나 소수를 저장합니다. 진행률, 우선순위, 점수 등에 활용됩니다.

priority: 1 progress: 75.5 rating: 4.8 version: 2.0

📅 날짜 (Date)

날짜와 시간 정보를 저장합니다. ISO 8601 형식을 권장합니다.

created: 2024-01-15 updated: 2024-01-20T14:30:00 due: 2024-02-01 published: 2024-01-15T09:00:00+09:00

☑️ 불리언 (Boolean)

참/거짓 값을 저장합니다. 상태 표시에 유용합니다.

completed: true published: false archived: false featured: true

📋 배열 (Array)

여러 값을 목록으로 저장합니다. 태그, 카테고리, 관련 항목 등에 사용됩니다.

tags: [프로그래밍, Python, 웹개발] categories: - 기술 - 튜토리얼 - 초급 authors: ["김작가", "이편집"]

🔗 링크 (Link)

다른 노트나 외부 URL을 참조합니다.

related: "[[관련 노트]]" source: https://example.com references: - "[[참고문헌 1]]" - "[[참고문헌 2]]"

🏷️ 객체 (Object)

구조화된 데이터를 중첩하여 저장합니다.

project: name: "웹 리뉴얼" team: "개발팀" budget: 50000000 deadline: 2024-03-31 author: name: "홍길동" email: "hong@example.com" role: "개발자"

🎨 별칭 (Aliases)

노트의 대체 이름을 지정합니다. 다양한 이름으로 링크할 수 있습니다.

aliases: - JS - 자바스크립트 - JavaScript ES6 - ECMAScript

🔧 커스텀 속성

필요에 따라 자유롭게 정의할 수 있습니다.

mood: "productive" weather: "sunny" location: "서울" energy_level: 8 word_count: 1523

🛠️ 메타데이터 빌더

아래 양식을 채워 자신만의 메타데이터를 생성해보세요.

생성된 메타데이터가 여기에 표시됩니다...

🔍 Dataview로 메타데이터 활용하기

Dataview 플러그인을 사용하면 메타데이터를 기반으로 동적인 뷰를 생성할 수 있습니다.

1. 상태별 작업 목록

```dataview TABLE status as "상태", priority as "우선순위", due as "마감일" FROM "Projects" WHERE status != "completed" SORT priority DESC, due ASC ```

결과 예시:

파일명 상태 우선순위 마감일
웹사이트 리뉴얼 진행중 높음 2024-02-15
API 문서화 검토중 보통 2024-02-20

2. 최근 수정된 노트

```dataview LIST FROM "" WHERE file.mtime >= date(today) - dur(7 days) SORT file.mtime DESC LIMIT 10 ```

지난 7일간 수정된 노트를 최신순으로 10개까지 표시합니다.

3. 태그별 통계

```dataview TABLE WITHOUT ID tag as "태그", length(rows) as "노트 수" FROM "" WHERE contains(tags, tag) FLATTEN tags as tag GROUP BY tag SORT length(rows) DESC ```

각 태그가 사용된 노트 수를 집계하여 표시합니다.

4. 진행률 추적

```dataview TABLE WITHOUT ID file.link as "프로젝트", progress as "진행률", "▓" * (progress/10) + "░" * (10 - progress/10) as "진행 상황" FROM "Projects" WHERE progress SORT progress DESC ```

프로젝트별 진행률을 시각적 바 형태로 표시합니다.

5. 커스텀 대시보드

```dataview TABLE WITHOUT ID "📅 " + dateformat(date, "MM-dd") as "날짜", "🏷️ " + join(tags, ", ") as "태그", "📊 " + status as "상태", "⚡ " + priority as "우선순위" FROM "Daily" WHERE date >= date(today) - dur(30 days) SORT date DESC ```

이모지를 활용한 시각적으로 풍부한 대시보드를 생성합니다.

✨ 메타데이터 활용 모범 사례

1. 일관된 스키마 유지

동일한 유형의 노트는 같은 메타데이터 구조를 사용하세요. 템플릿을 활용하면 일관성을 유지하기 쉽습니다.

💡 각 노트 유형별로 표준 템플릿을 만들고, Templater 플러그인으로 자동화하세요.

2. 의미 있는 속성명 사용

속성명은 직관적이고 명확해야 합니다. 약어보다는 전체 단어를 사용하세요.

✅ 좋은 예: created_date, project_status, priority_level ❌ 피해야 할 예: dt, stat, pri, p1

3. 적절한 데이터 타입 선택

각 속성에 맞는 데이터 타입을 사용하여 쿼리와 정렬이 올바르게 작동하도록 하세요.

  • 날짜: ISO 8601 형식 (YYYY-MM-DD)
  • 시간: 24시간 형식 (HH:MM)
  • 숫자: 따옴표 없이 입력
  • 불리언: true/false (소문자)

4. 선택적 속성 활용

모든 노트가 모든 속성을 가질 필요는 없습니다. 필수 속성과 선택적 속성을 구분하세요.

# 필수 속성 title: "제목" type: "note" created: 2024-01-25 # 선택적 속성 (필요시에만 추가) tags: [] priority: due_date:

5. 메타데이터 검증

잘못된 메타데이터는 쿼리 오류의 원인이 됩니다. 정기적으로 검증하세요.

⚠️ YAML은 들여쓰기에 민감합니다. 탭 대신 스페이스를 사용하고, 일관된 들여쓰기를 유지하세요.

6. 점진적 확장

처음부터 복잡한 스키마를 만들지 마세요. 필요에 따라 점진적으로 속성을 추가하세요.

  • 기본: title, tags, created
  • 확장: status, priority, category
  • 고급: custom fields, nested objects

🎯 실습 과제

메타데이터를 활용한 실제 시스템을 구축해보세요.

과제 1: 기본 메타데이터 추가 초급

기존 노트들에 기본적인 메타데이터를 추가해보세요.

  • 10개 이상의 노트 선택
  • 각 노트에 title, tags, created 추가
  • 일관된 태그 체계 구축
  • Dataview로 태그별 노트 목록 생성

과제 2: 프로젝트 관리 시스템 중급

메타데이터를 활용한 프로젝트 관리 시스템을 구축하세요.

  • 프로젝트 템플릿 생성
  • status, priority, due_date 등 속성 정의
  • 진행 상황 대시보드 생성
  • 마감일 기준 정렬 뷰 구현
  • 완료율 통계 표시

과제 3: 지식 관리 시스템 고급

복잡한 메타데이터 구조로 개인 지식 관리 시스템을 만드세요.

  • 계층적 분류 체계 설계
  • 학습 진도 추적 시스템
  • 복습 일정 자동 생성
  • 관련 개념 네트워크 구축
  • 이해도별 대시보드 생성

보너스 과제: 자동화 워크플로우 고급

Templater와 연동하여 메타데이터 자동화 시스템을 구축하세요.

  • 자동 날짜/시간 입력
  • 파일명 기반 메타데이터 추출
  • 조건부 메타데이터 생성
  • 일괄 메타데이터 업데이트 스크립트
  • 메타데이터 검증 시스템

❓ 자주 묻는 질문

YAML Front Matter가 제대로 인식되지 않아요

다음 사항들을 확인해보세요:

  • Front Matter는 파일의 맨 첫 줄부터 시작해야 합니다
  • 시작과 끝에 정확히 세 개의 대시(---)를 사용해야 합니다
  • 들여쓰기는 스페이스만 사용하세요 (탭 사용 금지)
  • 콜론(:) 뒤에는 공백이 있어야 합니다
  • 특수문자가 포함된 값은 따옴표로 감싸세요
Dataview 쿼리가 작동하지 않아요

Dataview 문제 해결 방법:

  • Dataview 플러그인이 활성화되어 있는지 확인
  • 쿼리 블록이 정확히 ```dataview로 시작하는지 확인
  • FROM 절의 경로가 올바른지 확인 (따옴표 필요)
  • 메타데이터 속성명의 대소문자가 정확한지 확인
  • 날짜 형식이 올바른지 확인 (YYYY-MM-DD)
메타데이터가 너무 많아지면 성능에 영향이 있나요?

일반적으로 메타데이터 자체는 성능에 큰 영향을 주지 않습니다. 하지만:

  • Dataview 쿼리가 복잡할수록 처리 시간이 늘어납니다
  • 수천 개의 파일을 스캔하는 쿼리는 느릴 수 있습니다
  • FROM 절에서 특정 폴더로 범위를 제한하면 성능이 향상됩니다
  • 인덱싱이 완료될 때까지 기다려야 할 수 있습니다
메타데이터를 일괄 수정하는 방법이 있나요?

여러 방법이 있습니다:

  • VSCode: 정규식을 사용한 찾기/바꾸기
  • Python 스크립트: 자동화된 일괄 처리
  • Templater: tp.file.cursor() 등을 활용한 반자동 처리
  • 외부 도구: sed, awk 등 커맨드라인 도구

⚠️ 일괄 수정 전에는 반드시 백업을 만드세요!
