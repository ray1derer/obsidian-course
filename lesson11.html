1. 핵심 플러그인 개요

코어 플러그인 vs 커뮤니티 플러그인

코어 플러그인

  • Obsidian 팀이 직접 개발
  • 안정성과 성능 보장
  • 기본 기능 확장
  • 정기 업데이트

커뮤니티 플러그인

  • 커뮤니티 개발자 제작
  • 다양한 특수 기능
  • 빠른 혁신
  • 주의 깊은 선택 필요

주의: 커뮤니티 플러그인은 설정에서 "안전 모드" 를 해제해야 사용할 수 있습니다.

2. 필수 코어 플러그인

일일 노트 (Daily Notes)

매일 자동으로 날짜별 노트를 생성

주요 설정
날짜 형식: YYYY-MM-DD
템플릿: Templates/일일노트
폴더: Daily Notes/{{date:YYYY/MM}}
일기 업무 로그 습관 추적

북마크 (Bookmarks)

중요한 노트, 검색, 그래프를 즐겨찾기

  • 파일, 폴더, 검색 쿼리 북마크
  • 그룹으로 구성
  • 빠른 접근 (Ctrl+Shift+B)

노트 작성기 (Note Composer)

노트 분할, 병합, 추출 기능

  • 긴 노트를 여러 파일로 분할
  • 선택 영역을 새 노트로 추출
  • 여러 노트 병합

아웃고잉 링크 (Outgoing Links)

현재 노트에서 링크된 모든 노트 표시

  • 링크 구조 한눈에 파악
  • 미해결 링크 확인
  • 빠른 탐색

3. 고급 코어 플러그인

캔버스 (Canvas)

무한한 캔버스에서 시각적으로 아이디어를 연결하고 구성

주요 기능

  • 노트를 카드로 배치
  • 화살표로 관계 표현
  • 이미지, 웹 콘텐츠 삽입
  • 색상과 그룹으로 구성

활용 예시

# 프로젝트 계획 캔버스
- 아이디어 브레인스토밍
- 프로세스 플로우 차트
- 마인드맵
- 스토리보드
- 연구 노트 연결

동기화 (Sync)

여러 기기 간 안전한 실시간 동기화

엔드투엔드 암호화

완전한 프라이버시 보장

버전 기록

1년간 모든 변경사항 추적

모바일 지원

iOS/Android 완벽 호환

선택적 동기화

폴더별 동기화 설정

5. 플러그인 설정 최적화

성능과 사용성 균형 맞추기

시작 속도 최적화

  • 필수 플러그인만 활성화
  • 무거운 플러그인은 필요시만 켜기
  • 플러그인 로딩 순서 조정

메모리 사용 관리

  • 중복 기능 플러그인 제거
  • 캐시 설정 최적화
  • 대용량 처리 플러그인 주의

설정 백업

  • .obsidian 폴더 정기 백업
  • 플러그인 설정 문서화
  • 설정 프로필 관리

추천 플러그인 설정 예시

{
  "daily-notes": {
    "folder": "Daily Notes",
    "format": "YYYY-MM-DD",
    "template": "Templates/Daily Note",
    "autorun": true
  },
  "templates": {
    "folder": "Templates",
    "dateFormat": "YYYY-MM-DD",
    "timeFormat": "HH:mm"
  },
  "file-recovery": {
    "interval": 5,
    "keepDays": 7
  }
}

6. 플러그인 문제 해결

일반적인 문제와 해결책

🔴 플러그인 충돌

증상: 특정 기능이 작동하지 않거나 오류 발생

해결:

  1. 안전 모드로 재시작
  2. 플러그인을 하나씩 활성화하며 테스트
  3. 충돌하는 플러그인 식별 및 대체

🟡 느린 성능

증상: Obsidian이 느리게 반응

해결:

  1. 개발자 도구로 성능 프로파일링
  2. 무거운 플러그인 비활성화
  3. 캐시 정리 및 재인덱싱

실습 프로젝트

🔧 나만의 플러그인 조합 만들기

자신의 워크플로우에 최적화된 플러그인 세트를 구성해봅시다.

Step 1: 워크플로우 분석

  • 주로 사용하는 기능 목록 작성
  • 불편한 점과 개선하고 싶은 부분 파악

Step 2: 플러그인 선택

  • 필요한 기능별 플러그인 조사
  • 사용자 리뷰와 업데이트 빈도 확인

Step 3: 설정 최적화

  • 각 플러그인 설정 커스터마이징
  • 단축키 충돌 해결

Step 4: 문서화

  • 사용 중인 플러그인 목록과 설정 기록
  • 워크플로우 가이드 작성

핵심 정리

목적에 맞는 선택

모든 플러그인이 필요한 것은 아님 - 워크플로우에 맞게 선택

점진적 도입

한 번에 하나씩 추가하며 충분히 익숙해진 후 다음 플러그인 도입

정기적 검토

사용하지 않는 플러그인은 비활성화하여 성능 유지
