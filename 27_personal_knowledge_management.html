들어가며

개인 지식 관리(Personal Knowledge Management, PKM)는 디지털 시대의 필수 역량이 되었습니다. 이번 강의에서는 옵시디언을 활용하여 효과적인 PKM 시스템을 구축하고, 지식을 체계적으로 관리하며 창의적으로 활용하는 방법을 학습합니다.

PKM의 핵심 가치

  • 외부 기억장치: 두 번째 뇌로서의 디지털 지식 저장소
  • 연결된 사고: 아이디어 간의 예상치 못한 연결 발견
  • 복합 학습: 수집, 처리, 연결, 창조의 순환
  • 장기적 성장: 시간이 지날수록 가치가 증가하는 지식 자산
  • 창의적 산출: 축적된 지식을 바탕으로 한 새로운 통찰

PKM 이론과 원칙

1. 지식 관리의 순환 구조

발견 (Discover)

정보를 찾고 수집하는 단계. 다양한 소스에서 가치 있는 정보를 포착합니다.

정제 (Distill)

수집한 정보를 분석하고 핵심을 추출하는 단계. 노이즈를 제거하고 본질을 파악합니다.

연결 (Connect)

정보들 간의 관계를 발견하고 연결하는 단계. 새로운 통찰이 생성됩니다.

창조 (Create)

축적된 지식을 바탕으로 새로운 아이디어나 콘텐츠를 만드는 단계입니다.

2. 제텔카스텐 원칙

# 제텔카스텐 핵심 원칙 ## 1. 원자적 노트 (Atomic Notes) - 하나의 노트는 하나의 아이디어만 담는다 - 독립적으로 이해 가능해야 한다 - 다른 노트와 자유롭게 연결될 수 있다 ## 2. 고유 식별자 (Unique Identifier) - 모든 노트는 고유한 ID를 가진다 - 예: 202401151430 (타임스탬프) - 또는: 1a1, 1a2, 1b1 (계층적 번호) ## 3. 연결의 원칙 (Linking Principle) - 관련 노트들을 적극적으로 연결한다 - 연결의 이유를 명시한다 - 예상치 못한 연결을 환영한다 ## 4. 지속적 발전 (Continuous Development) - 노트는 완성되지 않는다 - 새로운 통찰이 생기면 업데이트한다 - 시간이 지날수록 풍부해진다 ## 5. 자기 언어로 작성 (Own Words) - 단순 복사가 아닌 자신의 언어로 재구성 - 이해한 내용을 바탕으로 작성 - 미래의 자신이 이해할 수 있도록

3. PARA 방법론

# PARA 방법론 구조 ## 📁 Projects (프로젝트) 현재 진행 중인 작업들 - 명확한 결과물이 있음 - 마감일이 정해져 있음 - 활발히 작업 중 예시: - 📁 블로그 포스트 작성 - 📁 프레젠테이션 준비 - 📁 연구 논문 ## 📁 Areas (영역) 지속적으로 관리해야 하는 책임 영역 - 끝이 없는 지속적 관리 - 표준을 유지해야 함 - 정기적 검토 필요 예시: - 📁 건강 관리 - 📁 재정 관리 - 📁 가족 관계 ## 📁 Resources (자료) 미래에 유용할 수 있는 참고 자료 - 당장 사용하지 않음 - 나중에 참고할 가능성 - 주제별로 정리 예시: - 📁 프로그래밍 팁 - 📁 요리 레시피 - 📁 여행 정보 ## 📁 Archive (보관함) 더 이상 활성화되지 않은 항목들 - 완료된 프로젝트 - 비활성 영역 - 오래된 자료 예시: - 📁 2023년 프로젝트 - 📁 이전 직장 자료 - 📁 졸업 논문

4. CODE 방법

# CODE 방법: 정보를 지식으로 ## C - Capture (포착) 흥미롭고 유용한 정보를 빠르게 수집 ``` - 웹 클리퍼 사용 - 음성 메모 활용 - 스크린샷 저장 - 빠른 노트 작성 ``` ## O - Organize (조직) 수집한 정보를 체계적으로 정리 ``` - 폴더 구조 활용 - 태그 시스템 구축 - 메타데이터 추가 - 정기적 정리 ``` ## D - Distill (정제) 핵심 내용을 추출하고 요약 ``` - 프로그레시브 요약 - 핵심 문장 하이라이트 - 자신의 언어로 재작성 - 불필요한 내용 제거 ``` ## E - Express (표현) 학습한 내용을 창의적으로 활용 ``` - 블로그 포스트 작성 - 프레젠테이션 제작 - 새로운 프로젝트 시작 - 지식 공유 ```

// AI 강화 PKM 시스템 class AIEnhancedPKM { constructor(obsidianApp, aiService) { this.app = obsidianApp; this.ai = aiService; } // 자동 요약 생성 async generateSummary(notePath) { const content = await this.app.vault.read(notePath); const summary = await this.ai.summarize(content, { maxLength: 200, style: 'academic' }); // 프론트매터에 요약 추가 await this.updateFrontmatter(notePath, { summary: summary, summarizedAt: new Date().toISOString() }); } // 연결 제안 async suggestConnections(notePath) { const content = await this.app.vault.read(notePath); const embedding = await this.ai.getEmbedding(content); // 모든 노트의 임베딩과 비교 const allNotes = this.app.vault.getMarkdownFiles(); const similarities = []; for (const note of allNotes) { if (note.path === notePath) continue; const noteContent = await this.app.vault.read(note); const noteEmbedding = await this.ai.getEmbedding(noteContent); const similarity = this.cosineSimilarity(embedding, noteEmbedding); if (similarity > 0.7) { similarities.push({ note: note, similarity: similarity, reason: await this.ai.explainConnection(content, noteContent) }); } } return similarities.sort((a, b) => b.similarity - a.similarity); } // 질문 생성 async generateQuestions(notePath) { const content = await this.app.vault.read(notePath); const questions = await this.ai.generateQuestions(content, { types: ['clarification', 'extension', 'application'], count: 5 }); // 질문을 노트에 추가 const questionsSection = ` ## 🤔 탐구 질문 ${questions.map(q => `- ${q}`).join('\n')} `; await this.appendToNote(notePath, questionsSection); } // 개념 추출 async extractConcepts(notePath) { const content = await this.app.vault.read(notePath); const concepts = await this.ai.extractConcepts(content); // 각 개념에 대한 노트 생성 제안 for (const concept of concepts) { if (!await this.conceptNoteExists(concept)) { await this.createConceptNote(concept); } } } }
