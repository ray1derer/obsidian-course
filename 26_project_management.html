들어가며

이번 강의에서는 옵시디언을 활용하여 강력하고 유연한 프로젝트 관리 시스템을 구축하는 방법을 학습합니다. 개인 프로젝트부터 대규모 팀 프로젝트까지, 모든 규모의 프로젝트를 효과적으로 관리할 수 있는 체계를 만들어보겠습니다.

옵시디언 프로젝트 관리의 장점

  • 완전한 커스터마이징: 프로젝트 특성에 맞는 맞춤형 시스템
  • 연결된 정보: 모든 프로젝트 정보가 유기적으로 연결
  • 강력한 검색: 프로젝트 데이터의 즉각적인 검색과 필터링
  • 플러그인 생태계: 다양한 프로젝트 관리 플러그인 활용
  • 오프라인 작업: 인터넷 연결 없이도 완전한 기능 사용

주의사항: 옵시디언은 전통적인 프로젝트 관리 도구가 아니므로, 팀의 요구사항과 워크플로우에 맞게 시스템을 설계하는 것이 중요합니다.

# 태스크 대시보드 ## 🔴 긴급 태스크 ```dataview TABLE file.link as "태스크", assignee as "담당자", due as "마감일" FROM #task WHERE priority = "🔴 높음" AND status != "#done" SORT due ASC ``` ## 📅 오늘의 태스크 ```dataview TASK FROM #task WHERE contains(due, date(today)) GROUP BY file.link ``` ## 👤 담당자별 태스크 ```dataview TABLE WITHOUT ID assignee as "담당자", length(rows) as "총 태스크", length(filter(rows, (r) => r.status = "#doing")) as "진행중", length(filter(rows, (r) => r.status = "#done")) as "완료" FROM #task GROUP BY assignee ``` ## 📊 프로젝트별 진행률 ```dataviewjs const tasks = dv.pages("#task").where(t => t.project); const projects = tasks.groupBy(t => t.project); dv.table(["프로젝트", "전체", "완료", "진행률", "상태바"], projects.map(g => { const total = g.rows.length; const done = g.rows.filter(t => t.status === "#done").length; const percentage = Math.round((done / total) * 100); const progressBar = "█".repeat(Math.floor(percentage / 10)) + "░".repeat(10 - Math.floor(percentage / 10)); return [ g.key, total, done, percentage + "%", progressBar ]; }) ); ```

# 프로젝트 분석 쿼리 모음 ## 1. 지연된 태스크 찾기 ```dataview TABLE file.link as "태스크", assignee as "담당자", due as "마감일", (date(today) - due).days as "지연일" FROM #task WHERE status != "#done" AND due < date(today) SORT due ASC ``` ## 2. 담당자별 워크로드 분석 ```dataviewjs const members = dv.pages("#team-member"); const workload = []; for (const member of members) { const tasks = dv.pages("#task") .where(t => t.assignee && t.assignee.includes(member.file.name)); const totalHours = tasks .where(t => t.estimatedHours) .map(t => t.estimatedHours) .reduce((a, b) => a + b, 0); workload.push({ member: member.file.name, tasks: tasks.length, hours: totalHours, avgHours: (totalHours / tasks.length).toFixed(1) }); } dv.table(["팀원", "태스크 수", "총 시간", "평균 시간"], workload.map(w => [w.member, w.tasks, w.hours + "h", w.avgHours + "h"]) ); ``` ## 3. 스프린트 속도 분석 ```dataview TABLE WITHOUT ID sprint as "스프린트", planned as "계획 포인트", completed as "완료 포인트", round((completed / planned) * 100, 1) + "%" as "달성률" FROM "Projects/Sprints" WHERE file.name != "Sprint-Template" SORT sprint DESC LIMIT 10 ``` ## 4. 이슈 해결 시간 분석 ```dataviewjs const issues = dv.pages("#issue").where(i => i.resolvedDate); const resolutionTimes = issues.map(issue => { const created = moment(issue.createdDate); const resolved = moment(issue.resolvedDate); return { issue: issue.file.name, priority: issue.priority, days: resolved.diff(created, 'days'), category: issue.category }; }); // 우선순위별 평균 해결 시간 const byPriority = {}; ["🔴 높음", "🟡 보통", "🟢 낮음"].forEach(priority => { const filtered = resolutionTimes.filter(rt => rt.priority === priority); if (filtered.length > 0) { byPriority[priority] = (filtered.reduce((sum, rt) => sum + rt.days, 0) / filtered.length).toFixed(1); } }); dv.table(["우선순위", "평균 해결 시간"], Object.entries(byPriority).map(([priority, days]) => [priority, days + "일"]) ); ```
