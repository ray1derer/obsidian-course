그래프 뷰란?

그래프 뷰는 노트 간의 연결 관계를 시각적으로 표현하는 옵시디언의 핵심 기능입니다. 각 노트는 노드(점)로, 링크는 엣지(선)로 표현되어 지식의 네트워크를 한눈에 볼 수 있습니다.

그래프 뷰 종류

전역 그래프

전체 볼트의 모든 노트와 연결을 표시

  • 전체 지식 구조 파악
  • 주요 허브 노트 식별
  • 고립된 노트 발견
사용 시기: 전체적인 구조 분석, 정리가 필요한 영역 찾기

현재 노트를 중심으로 한 연결만 표시

  • 특정 주제 깊이 탐색
  • 직접 연결된 노트 확인
  • 컨텍스트 이해
사용 시기: 특정 노트의 맥락 파악, 관련 자료 탐색

  • 연결 기회 탐색

클러스터 분석

밀접하게 연결된 노트 그룹은 관련 주제나 프로젝트를 나타냅니다:

  • 밀집 클러스터: 깊이 있게 탐구한 주제
  • 느슨한 클러스터: 발전 가능성이 있는 영역
  • 브릿지 연결: 창의적 연결 가능성

실전 활용법

연구 주제 탐색

  1. 핵심 개념 노트에서 로컬 그래프 열기
  2. 연결된 노트들을 순차적으로 탐색
  3. 새로운 연결 가능성 발견
  4. 빈 공간에 새 노트 추가

노트 정리

  1. 전역 그래프에서 고립 노드 찾기
  2. 관련 있는 노트끼리 연결
  3. 불필요한 노트 아카이브
  4. 허브 노트 생성 고려

아이디어 개발

  1. 브레인스토밍 노트 생성
  2. 관련 개념들과 연결
  3. 그래프에서 패턴 관찰
  4. 새로운 연결점 발견

지식 네트워크 구축

다음 구조의 지식 네트워크를 만들어보세요:

  1. 중심 주제 노트 1개 생성 (MOC)
  2. 하위 주제 노트 5개 이상 생성
  3. 각 하위 주제에 세부 노트 2-3개 연결
  4. 교차 연결로 네트워크 강화
  5. 태그로 분류하고 색상 그룹 설정

완성 후 그래프 뷰에서 구조를 분석하고 개선점을 찾아보세요.

문제 해결

그래프가 너무 복잡해요

  • 필터를 사용하여 표시할 노트 제한
  • 깊이 설정을 낮춰서 근처 노드만 표시
  • 특정 태그나 폴더만 표시하도록 설정

노드가 겹쳐서 보여요

  • Force 설정에서 반발력 증가
  • 수동으로 노드 위치 조정
  • 확대하여 세부 영역 확인

성능이 느려요

  • 애니메이션 비활성화
  • 표시할 노드 수 제한
  • 하드웨어 가속 확인
