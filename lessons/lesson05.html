🏷️ 태그 vs 폴더: 패러다임의 전환

전통적인 파일 시스템에서 우리는 폴더에 파일을 "넣습니다". 하지만 현실에서 하나의 아이디어는 여러 범주에 속할 수 있습니다. 태그는 이런 다차원적 분류를 가능하게 합니다.

특징 폴더 태그
구조 계층적 (트리) 평면적 (네트워크)
위치 하나의 위치만 여러 태그 가능
변경 이동 시 링크 깨짐 위험 자유롭게 추가/제거
검색 경로 기반 태그 조합 검색
유연성 낮음 높음

핵심 원칙: 폴더는 "어디에 있는가?"를 정의하고, 태그는 "무엇인가?"를 정의합니다. 두 방식을 조합하면 최강의 조직 시스템이 됩니다.

🎨 태그 시스템 설계하기

1. 태그 문법과 규칙

Markdown 
# 기본 태그
#태그 #project #idea #todo

# 중첩 태그 (계층 구조)
#project/web #project/mobile
#book/fiction #book/nonfiction
#status/active #status/completed

# 다중 단어 태그
#machine-learning #data-science
#to-read #in-progress

# 특수문자 사용 불가 (이것들은 작동 안 함)
#태그 with spaces  ❌
#태그@특수문자     ❌
#123숫자시작       ❌

태그 작성 팁:
• 소문자 사용 권장 (일관성)
• 하이픈(-) 또는 언더스코어(_) 사용
• 단수형 사용 (#book not #books)
• 2-3단어 이내로 간결하게

3. 중첩 태그 전략

태그 계층 구조 예시 
#project
├── #project/active
│   ├── #project/active/website
│   ├── #project/active/mobile-app
│   └── #project/active/research
├── #project/planning
│   ├── #project/planning/2025
│   └── #project/planning/ideas
└── #project/completed
    ├── #project/completed/2024
    └── #project/completed/2023

#learning
├── #learning/programming
│   ├── #learning/programming/python
│   ├── #learning/programming/javascript
│   └── #learning/programming/rust
├── #learning/languages
│   ├── #learning/languages/english
│   └── #learning/languages/japanese
└── #learning/skills
    ├── #learning/skills/writing
    └── #learning/skills/design

주의: 태그 계층을 너무 깊게 만들지 마세요. 일반적으로 2-3단계가 적절합니다. 더 깊어지면 폴더를 사용하는 것이 낫습니다.

📋 속성(Properties) - 구조화된 메타데이터

옵시디언의 속성 기능은 노트에 구조화된 메타데이터를 추가할 수 있게 해줍니다. YAML 프론트매터를 사용하며, 검색과 필터링에 강력한 도구가 됩니다.

1. 기본 속성 사용법

속성 편집기

Properties

project meeting planning

2. YAML 프론트매터 문법

YAML 
---
# 텍스트 속성
title: "프로젝트 킥오프 미팅"
author: "김철수"
description: "새 프로젝트 시작을 위한 전체 팀 미팅"

# 날짜/시간
date: 2025-06-29
created: 2025-06-29T10:00:00
modified: 2025-06-29T15:30:00
due: 2025-07-15

# 태그 (리스트)
tags: [project, meeting, planning, important]
categories: ["업무", "프로젝트 관리"]

# 상태 및 우선순위
status: in-progress
priority: high
completion: 25

# 불린(Boolean) 값
published: false
archived: false
pinned: true

# 숫자
version: 1.2
rating: 4.5
word-count: 1523

# 링크
related: 
  - "[[프로젝트 계획서]]"
  - "[[팀원 연락처]]"
  - "[[예산 문서]]"

# 사용자 정의 속성
project-code: "PRJ-2025-001"
client: "ABC Company"
budget: "$50,000"
team-members: ["김철수", "이영희", "박민수"]
---

🔍 태그와 속성을 활용한 검색 전략

1. 기본 검색 문법

Search Query 
# 태그 검색
tag:#project
tag:#project/active
tag:#meeting OR tag:#call

# 속성 검색
[status:completed]
[priority:high]
[date:2025-06-29]
[progress:>50]

# 복합 검색
tag:#project AND [status:active]
tag:#learning [priority:high] Python

# 부정 검색
-tag:#archived
-[status:completed]

# 정규식 검색
/tag:#project\/.*/

2. 고급 필터링

검색 쿼리 예시

시나리오 1: 이번 주 완료하지 못한 높은 우선순위 작업 찾기

tag:#task -[status:completed] [priority:high] [due:<=2025-07-05]

시나리오 2: 특정 프로젝트의 회의록 찾기

tag:#meeting [project-code:PRJ-2025-001] [date:>=2025-06-01]

시나리오 3: 리뷰가 필요한 초안 문서

[status:draft] [review-needed:true] -tag:#personal

3. 검색 결과 저장

자주 사용하는 검색 쿼리는 노트로 저장하여 재사용할 수 있습니다.

Markdown 
# 🔍 저장된 검색 쿼리

## 📋 업무 관련
- **오늘 할 일**: `tag:#today -[status:completed]`
- **이번 주 마감**: `[due:<=2025-07-05] -[status:completed]`
- **검토 대기**: `[status:review] OR [review-needed:true]`

## 📚 학습 관련
- **진행 중인 강의**: `tag:#course [status:in-progress]`
- **읽을 책 목록**: `tag:#book [status:to-read]`
- **복습 필요**: `tag:#study [review-date:<=2025-06-29]`

## 🗂️ 프로젝트
- **활성 프로젝트**: `tag:#project/active`
- **대기 중인 작업**: `[status:waiting] [assigned-to:me]`
- **완료된 마일스톤**: `tag:#milestone [status:completed]`

💡 태그 시스템 설계 베스트 프랙티스

1. 태그 명명 규칙

✅ 좋은 예시

  • ✓ #project-management
  • ✓ #book/fiction
  • ✓ #status/in-progress
  • ✓ #meeting-2025-06
  • ✓ #priority-high

❌ 피해야 할 예시

  • ✗ #Project Management (공백)
  • ✗ #book/소설 (언어 혼용)
  • ✗ #진행중 (너무 일반적)
  • ✗ #mtg (불명확한 약어)
  • ✗ #!important! (특수문자)

2. 태그 계층 구조 설계

설계 원칙:
1. 상위 태그는 포괄적이고 안정적이어야 함
2. 하위 태그는 구체적이고 실용적이어야 함
3. 3단계 이상 깊어지면 재고려 필요
4. 사용 빈도가 낮은 태그는 제거

3. 태그 vs 속성 선택 가이드

사용 사례 태그 사용 속성 사용
카테고리 분류 ✅ ⚪
날짜/시간 정보 ⚪ ✅
진행 상태 🔵 ✅
수치 데이터 ⚪ ✅
빠른 필터링 ✅ 🔵

✅ 권장 | 🔵 상황에 따라 | ⚪ 비권장

📊 태그 분석과 유지보수

1. 태그 사용 현황 분석

Dataview 플러그인을 활용하여 태그 사용 현황을 분석할 수 있습니다.

Dataview Query 
```dataview
TABLE WITHOUT ID
  tag as "태그",
  length(rows) as "사용 횟수"
FROM ""
FLATTEN file.tags as tag
WHERE tag != null
GROUP BY tag
SORT length(rows) DESC
LIMIT 20
```

2. 태그 정리 체크리스트

월간 태그 시스템 점검

  • 사용하지 않는 태그 식별 및 제거
  • 유사한 태그 통합 (예: #todo와 #task)
  • 잘못된 철자의 태그 수정
  • 새로운 카테고리 필요성 검토
  • 태그 문서 업데이트
  • 팀원과 태그 규칙 공유 (팀 프로젝트의 경우)

3. 태그 마이그레이션

태그 이름 변경 시 주의사항:
1. Tag Wrangler 플러그인 사용 권장
2. 변경 전 전체 백업 필수
3. 점진적으로 변경 (한 번에 너무 많이 변경하지 않기)
4. 변경 내역 문서화

다음 강의 예고:
